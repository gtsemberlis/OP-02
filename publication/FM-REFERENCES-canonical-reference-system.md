# Canonical Reference System

## Section Purpose

Canonical canonical reference system for OP-01.

## Canonical Content

### Purpose

Canonical references organize recurring people, places, scenes, artifacts, appendices, and publications. They are editorial tools for the Game Master and never appear within the fiction.

### Reference Prefixes

| Prefix | Category | Example |
|---|---|---|
| CB | Campaign Bible | CB-01 |
| OP | Operation Book | OP-01 |
| GMC | GM Console | GMC-01 |
| S | Scene | S-07 |
| N | NPC | N-03 |
| L | Location | L-02 |
| H | Handout / Historical Artifact | H-06 |
| A | Appendix | A-B |

### Stability Rules

- Canonical references are permanent.
- Existing references are never reassigned.
- Numbers are never reused.
- Cross-references use canonical identifiers rather than page numbers.
- Object UUIDs remain stable even if titles or presentation change.

### Organizational Principle

Canonical references organize information. They do not indicate narrative importance.

Their purpose is to reduce cognitive load and support navigation across every derived publication.
